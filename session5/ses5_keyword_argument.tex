\section{Working with Keyword Arguments}
Session 5 keeps the same prediction rule from Session 4 and the same beginner-friendly flow that later becomes a class in Session 6. We are not adding a new classifier idea here. We are only changing how the same code is called from outside the functions.

\subsubsection{Advantages of Keyword Arguments}
There are several advantages:
\begin{itemize}\setlength{\itemsep}{0pt}
    \item The function call becomes easier to read.
    \item The meaning of each value becomes visible at the call site.
    \item Default values make a function easier to reuse.
\end{itemize}

\subsubsection{Keyword Argument Syntax}
When you use keyword arguments, you write the parameter name, then an equal sign, then the value. For example:
\begin{center}
    \texttt{threshold=1.8}
\end{center}

\noindent
Because the parameter name is written in the call, a reader can understand the call more quickly.

\subsubsection{Combining Positional and Keyword Arguments}
Python also allows you to combine both styles in one function call. The positional arguments must come first, and the keyword arguments must come after them.

\begin{center}
    \texttt{run\_prediction\_loop(dataset, threshold=1.8, print\_each=False)}
\end{center}

\noindent
In that example:
\begin{itemize}
    \item \texttt{dataset} is passed positionally,
    \item \texttt{threshold=1.8} is passed by keyword,
    \item \texttt{print\_each=False} is passed by keyword.
\end{itemize}

\subsection{Task 1: Refactor \texttt{calculate\_accuracy} to use keyword-friendly defaults}
This is the smallest refactor in the session, but it is a very important one. The calculation stays exactly the same. Only the function interface changes.

\begin{tcolorbox}[title=Task 1: Refactor \texttt{calculate\_accuracy} to use keyword-friendly defaults]
Work only in \texttt{session5/session5\_iris\_template.py}. Do not create a new file, and do not rename the function.

\begin{enumerate}
    \item Find the comment \texttt{\# Task 1: Refactor calculate\_accuracy to use keyword-friendly defaults}. It is directly above the function \texttt{calculate\_accuracy(...)}.
    \item In the function header, replace the \texttt{<your\_code>} after \texttt{correct=} with \texttt{0}.
    \item Replace the \texttt{<your\_code>} after \texttt{total=} with \texttt{0}.
    \item After both replacements, the full function header should become:
    \texttt{def calculate\_accuracy(correct=0, total=0):}
    \item Keep the calculation logic inside the function body. Do not change the \texttt{if total > 0} logic.
    \item Now move to the comment \texttt{\# Task 2: Rename setup\_application to run\_classifier\_pipeline}.
    \item Inside that wrapper function, find the call to \texttt{calculate\_accuracy(...)}.
    \item Change that call so it uses keyword arguments:
    \texttt{correct=correct} and \texttt{total=total}.
\end{enumerate}
\end{tcolorbox}

Use this exact function shape as the target:

\begin{minted}[fontsize=\small, linenos, breaklines=true, frame=lines]{python}
def calculate_accuracy(correct=0, total=0):
    if total > 0:
        accuracy = (correct / total) * 100
    else:
        accuracy = 0.0

    return accuracy
\end{minted}

\subsection{Task 2: Rename \texttt{setup\_application} to \texttt{run\_classifier\_pipeline}}
The old name \texttt{setup\_application} is too broad for beginners because the function does more than setup. It builds the dataset, runs the prediction loop, and computes the final accuracy. A clearer name helps students understand the purpose of the function.

\begin{tcolorbox}[title=Task 2: Rename \texttt{setup\_application} to \texttt{run\_classifier\_pipeline}]
Work only in \texttt{session5/session5\_iris\_template.py}.

\begin{enumerate}
    \item Find the comment \texttt{\# Task 2: Rename setup\_application to run\_classifier\_pipeline}. It is directly above the function definition.
    \item Notice that the template still uses the old name \texttt{setup\_application(...)}.
    \item Rename the function from \texttt{setup\_application(...)} to \texttt{run\_classifier\_pipeline(...)}.
    \item In the function header, replace the \texttt{<your\_code>} after \texttt{threshold=} with \texttt{2.0}.
    \item Replace the \texttt{<your\_code>} after \texttt{print\_each=} with \texttt{True}.
    \item After both replacements, the full function header should become:
    \texttt{def run\_classifier\_pipeline(threshold=2.0, print\_each=True):}
    \item Keep the three main steps inside this function:
    \begin{itemize}
        \item build the dataset using \texttt{setup\_application\_list()},
        \item run the prediction loop using \texttt{run\_prediction\_loop(...)},
        \item compute accuracy using \texttt{calculate\_accuracy(...)}.
    \end{itemize}
    \item If you still see the old name anywhere else later, replace it there too.
    \item Keep the final return line:
    \texttt{return correct, wrong, total, y\_pred\_list, accuracy}
\end{enumerate}
\end{tcolorbox}

Use this exact function shape as the target:

\begin{minted}[fontsize=\small, linenos, breaklines=true, frame=lines]{python}
def run_classifier_pipeline(threshold=2.0, print_each=True):
    dataset = setup_application_list()
    correct, wrong, total, y_pred_list = run_prediction_loop(
        dataset,
        threshold,
        print_each,
    )
    accuracy = calculate_accuracy(
        correct=correct,
        total=total,
    )
    return correct, wrong, total, y_pred_list, accuracy
\end{minted}

\begin{tcolorbox}[title=Checkpoint]
After Task 2, students should be able to explain:
\begin{itemize}
    \item \texttt{threshold=2.0} gives a default threshold,
    \item \texttt{print\_each=True} means each sample is printed by default,
    \item and \texttt{run\_classifier\_pipeline(...)} now wraps the same Session 4 steps in one reusable function.
\end{itemize}
\end{tcolorbox}
